# 00 · 七阶段流水线：完成标准与失败模式

> 本文是 SKILL.md「每阶段完成标准 / 失败模式」的扩展位。
> 解决两个工程问题：①**什么时候算做完**（完成标准 / Definition of Done）；
> ②**做完但做错了怎么办**（失败模式 + 症状 + 应对，防御性设计）。
> 用法：每阶段动手前读「完成标准」清单，收尾前逐条勾选；卡住时查「失败模式」对照症状取应对。

---

## 1. 七阶段全景图

```
零 预处理决策   判格式 + 冲突检测 + 搜更优源        先判断，再动手
   │  raw 书（PDF/扫描/EPUB/MD/TXT）
   ▼
一 结构化       → book_structured.md（章节边界 + 全角→半角 + 归档 raw）
   │  full_text.txt → book_structured.md
   ▼
二 深度解析     多 Agent 并行 · 四层提取（概念L1→框架L2→方法L3→原则L4）
   │  四层知识 JSON + 隐含假设 + 原则冲突调和
   ▼
三 认知架构     五要素（身份 / 知识分配 / 能力边界 / 行为模式 / 交互协议）
   │  认知架构决策表
   ▼
四 Skill 编写   SKILL.md(<500行) + references 六件套 + scripts + 路由加载指令
   │  SKILL.md + references/*
   ▼
五 五维验证     覆盖度 / 关系 / 应用 / 判断 / 边界——以"用"为验，不以"记"为验
   │  五维验证报告
   ▼
六 归档闭环     知识图谱 + 工作日志 + 索引更新（原点 KB）
```

---

## 2. 阶段间数据流（产物链）

| 阶段 | 输入 | 产物（交付物） | 落地位置 |
|------|------|---------------|---------|
| 零 | raw 书 | 格式判定结论 + 冲突检测结论 + 源文件选定 | 决策记录（内存/草稿） |
| 一 | raw 书 → `full_text.txt` | `book_structured.md`（章节边界确认） | skill 工作区；raw 归档原点 |
| 二 | `book_structured.md` | 四层知识 JSON（含隐含假设 + 冲突调和） | Agent 产出汇总 |
| 三 | 四层知识 | 认知架构五要素答案表 | 设计草稿 |
| 四 | 认知架构 + 四层知识 | `SKILL.md` + `references/` 六件套 + `scripts/` | `~/.claude/skills/book-*/` |
| 五 | 产出 skill | 五维验证报告（含案例锚点检查） | 验证记录 |
| 六 | 全部产物 | 工作日志 + `knowledge_graph.md` + 索引条目 | 原点 KB |

> 一条断链 = 下游全废：阶段一不归档 raw，阶段六无法回溯；阶段四不写路由加载指令，阶段五「应用维」必挂。

## 3. 每阶段完成标准（checkbox：什么时候算做完）

### 阶段零 · 预处理决策
- □ 冲突检测完成（已搜 `~/.claude/skills/` 同名/近似 skill 并定下「更新/新建/取消」）
- □ 格式判定完成（**文件名标 (OCR) ≠ Read 能读**，已验证是否有可提取文字层）
- □ 更优源文件已搜索（优化过的 MD > 学习成长目录素材 > 已有 book-* 预处理产物）

### 阶段一 · 结构化
- □ 输出 `book_structured.md`
- □ 章节边界确认（精确行号写入，配合 `scripts/detect_structure.py`）
- □ 全角 → 半角完成（标点/字母/数字）
- □ 已归档原点 raw（`full_text.txt` 与原始书留痕，可回溯）

### 阶段二 · 深度解析
- □ Agent prompt 含「验证锚点」要求（每个知识要素须标书中出处）
- □ 亲自读 ≥2 个核心章交叉验证（不依赖 Agent 单方输出）
- □ 产出含隐含假设 + 原则冲突调和（不只罗列，要调和矛盾）
- □ 四层齐全（概念 L1 / 框架 L2 / 方法 L3 / 原则 L4 无缺层）

### 阶段三 · 认知架构
- □ 五要素已答：身份定位 / 知识分配 / 能力边界 / 行为模式 / 交互协议

### 阶段四 · Skill 编写
- □ `SKILL.md` < 500 行，且 YAML frontmatter 用 block scalar `|`
- □ references 六件套齐全（concepts / frameworks / methods / principles / cases / knowledge_graph）
- □ 每条路由带**具体加载指令**（格式 `方法→references/methods.md#方法N`）

### 阶段五 · 五维验证
- □ 五维全过：覆盖度 / 关系 / 应用 / 判断 / 边界（含案例锚点检查）

### 阶段六 · 归档闭环
- □ 工作日志已追加（append-only）
- □ 知识图谱已生成（`knowledge_graph.md`，无孤立节点）
- □ 索引已更新（原点 KB 可检索到本 skill）

## 4. 每阶段失败模式 + 症状 + 应对（做完但做错了怎么办）

### 阶段零
| 失败模式 | 症状 | 应对 |
|---------|------|------|
| 误判扫描版为文本版 | `pdftotext` 输出为空/乱码/全是页码 | 改用 PyMuPDF(`scripts/extract_pdf.py`) 提文字层；仍无 → tesseract OCR |
| 漏做冲突检测就新建 | 后期发现已有同名 skill，工作白做或覆盖他人成果 | 动手前必搜 `~/.claude/skills/`；已存在且质量高 → 仅补路由加载指令不推翻 |
| 忽略更优源 | 从扫描版苦干，其实已有优化 MD | 按优先级搜源；找到优质 MD 直接跳到阶段一 |

### 阶段一
| 失败模式 | 症状 | 应对 |
|---------|------|------|
| 章节边界识别错 | 目录/正文错位，章数对不上 | 宽匹配 + 多候选行 + 人工确认行号后写入 |
| 全角半角混用残留 | 后续 Agent 解析串字、检索命中率低 | 脚本转标点/字母/数字；中文正文交 Agent 智能识别，勿暴力替换 |
| raw 未归档 | 阶段六无法回溯、无法复现 | 结构化前先把 `full_text.txt` 与原书归档原点 |

### 阶段二
| 失败模式 | 症状 | 应对 |
|---------|------|------|
| Agent 遗漏核心概念 | 亲自交叉验证发现缺失 | 派独立 Agent 补解析该章 |
| 知识要素无出处锚点 | 阶段五案例锚点检查失败 | prompt 强制「每要素标书中章节/页」；缺则回炉补 |
| 只罗列不调和矛盾 | 原则之间互相打架，skill 给不出一致判断 | 显式写「原则冲突调和」段，定优先级/边界条件 |
| 缺层（只到方法不到原则） | 四层验收不齐 | 补 L4 原则层（边界条件 + 适用前提） |

### 阶段三
| 失败模式 | 症状 | 应对 |
|---------|------|------|
| 身份定位摇摆 | 一会儿作者口吻一会儿旁观讲解，语气分裂 | 五要素先定身份并贯穿全 skill |
| 知识分配失衡 | SKILL.md 塞满细节超 500 行 | 细节下沉 references，SKILL.md 只留路由 + 判断骨架 |

### 阶段四
| 失败模式 | 症状 | 应对 |
|---------|------|------|
| 路由「写了调不到」 | references 有内容，但 SKILL 无加载指令 | 每条路由补加载指令 `X→references/Y.md#锚点`（写了调不到=知识不存在）|
| SKILL.md 超 500 行 | 行数门禁不过 | 把概念/框架/方法/案例正文下沉到 references |
| 六件套缺件 | `check_skill.py` 报缺文件 | 补齐缺失件；空件也算不齐 |

### 阶段五
| 失败模式 | 症状 | 应对 |
|---------|------|------|
| 验证只测「记得住」 | 概念全对但变体场景判断错 | 以用为验：跑应用/判断/边界三维角色扮演 |
| 案例锚点缺失 | 应用维角色扮演无出处可对照 | 回阶段二补锚点，再复验 |
| 边界场景不会说「不」 | 越界请求被照单全收 | 在 principles.md 固化能力边界，复测边界三场景 |

### 阶段六
| 失败模式 | 症状 | 应对 |
|---------|------|------|
| 日志未追加 | 复盘无痕、无法迭代 | 收尾即 append 工作日志（含本轮决策与坑） |
| 知识图谱有孤立节点 | `knowledge_graph.md` 关系维不过 | 补全节点关系，消灭孤立点 |
| 索引未更新 | 原点 KB 检索不到，等于没归档 | 更新索引条目，确认可被检索到 |

---

> 工程要点：**每阶段的产物即下一阶段的输入**，断一环则下游全废；
> 完成标准是「能否进入下一阶段」的闸门，失败模式是「闸门已过但暗藏缺陷」的探针——两者都查，才算工程级闭环。
